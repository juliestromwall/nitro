Your Onboarding Checklist

What you need to do to add a collaborator to RepCommish

Six steps. You can finish all the required ones in about 15 minutes once you've got your buddy's GitHub username.

Get from your buddy first

  • His GitHub username (or the email he uses on GitHub) — for the repo invite
  • His email address — only if he's getting Supabase access (optional)
  • Confirmation he has a Mac (the setup guide is Mac-specific)

1Commit and push your current work

You have uncommitted local changes — the entire Brand Admin Payments tracker (PaymentsTracker page, demo data, Tony portal mode). If your buddy clones GitHub right now, he won't see any of it.

From your terminal:

cd ~/Projects/nitro
git status

You'll see modified and untracked files. Commit only the code — skip the marketing PDFs, sales decks, and one-sheeter HTMLs that are personal working files:

git add src/pages/PaymentsTracker.jsx \
        src/lib/paymentsDemoData.js \
        src/layouts/AppLayout.jsx \
        src/pages/CompanyDetail.jsx \
        src/components/company/CompanyCommission.jsx \
        setup-guide.html \
        your-checklist.html

git commit -m "Add Brand Admin payments tracker + onboarding docs"
git push
Want Claude to do this for you? In your Claude Code session just say "commit and push the in-progress work — code only, skip the marketing files."

2Add him as a GitHub collaborator

  1. Go to github.com/juliestromwall/nitro/settings/access
  2. Click Add people
  3. Type his GitHub username or email, pick Write role (lets him push but not change repo settings)
  4. Click Add to repository

He'll get an invite email. He has to click Accept before he can push.

3Send him the .env file (securely)

Your .env file at ~/Projects/nitro/.env has Supabase API keys. He needs an exact copy of it.

Do not send it over email, Slack, iMessage, or any chat app. Those leak. Anyone who gets the file can read/write the production database.

Pick one of these:

MethodHow
1PasswordCreate a Secure Note or Document item, attach the file, share to him by email. He needs a 1Password account.
Bitwarden SendFree. Upload the file at send.bitwarden.com, set a password and expiry, share the link separately from the password.
SignalEnd-to-end encrypted. Attach the file in a Signal chat with him.
AirDropIf you're physically near him. Show hidden files in Finder first with ⌘⇧. so you can see .env.

Where he'll put it

He puts the file at ~/Projects/nitro/.env on his machine — same folder as package.json.

4Send him the setup guide

The setup guide is already saved at ~/Projects/nitro/setup-guide.html. Send it to him one of these ways:

  • AirDrop — if he's nearby
  • Email attachment — the guide itself has no secrets, so email is fine
  • GitHub Gist — for a shareable link: 
    brew install gh
gh auth login   # one-time
gh gist create ~/Projects/nitro/setup-guide.html --public
    You'll get a URL like gist.github.com/… — send him that.
  • Or it's already in the repo — once you push (step 1), he can open it at ~/Projects/nitro/setup-guide.html right after he clones

5Add him to Supabase (optional)

Skip this unless he'll be editing database tables, RLS policies, or deploying edge functions.

  1. Go to supabase.com/dashboard
  2. Open the RepCommish project
  3. Settings → Team (in the left sidebar)
  4. Invite a member, enter his email, role Developer (read + write but not billing)

6Hostinger VPS access (optional)

Skip this unless he'll be deploying to production (repcommish.com). Most development happens locally, so most people never need this.

If he does need it, the easy path: ask him to send you the contents of his public SSH key (~/.ssh/id_ed25519.pub on his Mac), then SSH into your VPS and append it to ~/.ssh/authorized_keys. He never gets your private key — only his own public key gets added.

Quick recap — what to send him

  • GitHub invite (auto-sent when you add him in step 2)
  • The .env file via 1Password / Signal / AirDrop (step 3)
  • The setup-guide.html file (step 4)
  • Supabase invite if applicable (step 5)
When he's done with the setup guide, he'll be running npm run dev locally and seeing the demo account — the same view you have. You can then collaborate via normal git pushes and pulls.
