How to Share Your Local Server

Let Tony (or anyone) click through your in-progress work without deploying first

The site running at localhost:5174 on your laptop is invisible to everyone else — even on the same Wi-Fi. To let someone else see it, you need to bridge it to the internet temporarily. Three options, ranked best to worst for your situation.

1Cloudflare Tunnel — the recommended path

Cloudflare gives you a free, instant public URL that points to your localhost:5174. Tony opens that URL in any browser, anywhere — and as you edit code on your machine, he sees the changes live. Like magic.

One-time install

brew install cloudflared

Takes about 20 seconds. You only do this once, ever.

Every time you want to share

First — in one terminal window, start your dev server like usual:

cd ~/Projects/nitro
npm run dev

It'll print http://localhost:5174/. Leave that terminal open.

Then — open a second terminal window (Cmd+T in Warp or Terminal.app) and run:

cloudflared tunnel --url http://localhost:5174

Within a few seconds it'll print something like:

Your quick Tunnel has been created!
Visit it at:
https://elephant-piano-grass.trycloudflare.com

Text that URL to Tony. He opens it in any browser — he's now looking at your dev server.

Stopping the share: press Ctrl+C in the cloudflared terminal. The URL stops working immediately. Tony can't get back in until you start a new tunnel.

Things to know

  • The URL changes every time you start a new tunnel. Don't bookmark it — just send a fresh link each session.
  • If your laptop sleeps or you close the cloudflared terminal, the tunnel dies. Tony's connection drops.
  • The free version supports a generous amount of traffic for demos. You won't hit limits with one viewer.
  • Anyone who has the URL can access it — treat it like an unlisted YouTube link, not a password-protected page.
Critical to understand: Tony is looking at your machine. He's logged in as whoever you're logged in as. He shares the same Supabase database. If he adds an entry or changes a setting, it's a real change in your real database. Don't share the tunnel for anything where a stranger might land on the URL — only with Tony.

2Screen share

Lowest-effort option. Tony doesn't need a URL, doesn't need to install anything, doesn't even need a Mac.

  • Zoom — "Share Screen" button at the bottom. Pick your browser window. Tony sees what you see.
  • Google Meet — same idea. "Present now" → window.
  • Apple's screen share — if he has a Mac and you're both signed into iMessage, you can right-click his name in Messages → "Share My Screen." No third party.

Downside: Tony can only watch, not click. If he wants to interact, jump to option 1 instead.

3Tony runs his own copy

If Tony's actually going to write code, he should run his own local server. He goes through the full setup once and then has his own localhost:5174 on his machine.

Send him setup-guide.html — it walks him through it.

Once he's set up, you collaborate via GitHub (you push, he pulls, vice versa). The Cloudflare tunnel is no longer necessary because he doesn't need to see your machine — he sees his own.
